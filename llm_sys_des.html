LLM — System Design Framework

Copy Notion template ↗

Most effective when the system requires language understanding, reasoning, or generation beyond deterministic logic.

Problem

Define the user problem and why an LLM is needed.

  • User problem
  • Primary user
  • Job to be done
  • Why current solutions fail
  • Why LLM is required

LLM Role

Define what the LLM is responsible for.

  • Task type (summarize, explain, compare, etc.)
  • LLM responsibility
  • Non-LLM responsibility
  • Output type (text / structured)

Inputs & Context

Define what the model sees.

  • User input
  • Structured system data
  • Business rules
  • Session context
  • Static vs dynamic context

Prompt & Generation

Define how the LLM is instructed.

  • System instruction
  • Task instruction
  • Context injection
  • Constraints
  • Output schema

Output & UX

Define what the user sees.

  • Response format
  • Level of detail
  • Regenerate / refine
  • UI placement
  • Trust signals

Guardrails

Control safety and reliability.

  • Input validation
  • Prompt constraints
  • Output moderation
  • Hallucination control
  • Fallback logic

Evaluation

Measure system performance.

  • Quality (relevance, correctness)
  • Product metrics (usage, satisfaction)
  • System metrics (latency, cost)
  • Offline evals
  • Online experiments

Feedback Loop

Improve system over time.

  • User feedback signals
  • Failure analysis
  • Prompt iteration
  • Monitoring
  • Continuous improvement

When to use

Works well for

  • Explanation systems
  • Summarization tools
  • AI copilots
  • Comparison engines
  • Writing assistants
  • Insight generation layers

Not ideal for

  • Pure prediction systems (use ML framework)
  • Knowledge-heavy QA (use RAG framework)
  • Multi-step tool execution (use Agent framework)
  • Deterministic pipelines
