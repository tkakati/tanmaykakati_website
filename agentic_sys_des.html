Agentic System Design Framework

Most effective when systems must make sequential decisions, adapt to feedback, and operate under uncertainty.

Objective

Define what the agent is trying to achieve.

  • Task definition
  • Success metric
  • Stop criteria
  • Constraints

Planner

Define how the system decides what to do next.

  • Decision policy (deterministic vs LLM)
  • Explore vs exploit strategy
  • Iteration limits
  • Routing logic

Tools & Actions

Define how the agent interacts with the world.

  • Retrieval systems
  • External APIs
  • Execution actions
  • Tool selection logic

State & Memory

Define what the system remembers.

  • Session state
  • Long-term memory
  • Past actions / queries
  • User-specific memory

Evaluation

Define how performance is measured.

  • Output quality
  • Product metrics
  • System metrics
  • Thresholds for continuation

Guardrails

Define safety and reliability.

  • Input validation
  • Output validation
  • Failure modes
  • Fallback logic

Output & UX

Define what the user sees.

  • Output format
  • Actionability
  • Latency handling
  • Progressive disclosure

Feedback Loop

Define how the system improves.

  • User signals
  • Performance tracking
  • Iteration logic
  • Continuous improvement

When to use

Works well for

  • Multi-step decision systems
  • Retrieval + reasoning workflows
  • Exploration-heavy problems
  • Systems with feedback loops

Not ideal for

  • Static pipelines
  • Pure prediction systems
  • Simple CRUD workflows
  • Fully deterministic tasks

Applied in practice

HireFeed — Agentic Hiring Post Discovery

Agentic system that plans search, retrieves hiring signals, evaluates quality, and iterates until enough actionable opportunities are found.
