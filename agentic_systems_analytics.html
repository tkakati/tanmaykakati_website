Agentic Systems Analytics Framework

Copy Notion template ↗

A structured approach to measuring decision quality, loop efficiency, reliability, and outcome alignment in agentic systems.

Most effective when linking system state → decisions → execution → outcomes.

1. Objective

  • Define the system outcome that matters most.
  • Set one primary metric for value delivery.
  • Use secondary, diagnostic, and guardrail metrics to explain movement.

2. System Loop Model

Stage What happens Why it matters
PlannerTransforms state and goals into an executable plan.Sets direction and limits wasted actions.
ExecutionRuns tools, APIs, and actions defined by the plan.Turns decisions into measurable system behavior.
EvaluationChecks outputs against quality, policy, and task criteria.Separates useful outputs from noisy ones.
DecisionSelects next action: continue, revise, retry, or stop.Controls loop quality, speed, and cost.
LoopUpdates state and repeats until stop conditions are met.Determines final outcome consistency over time.
System state → Planner → Execution → Evaluation → Decision → Loop

3. Core Dimensions

Dimension What it measures Example questions
Planner IntelligenceQuality of plan selection under constraints.Are plans coherent and goal-aligned on first pass?
Execution EfficiencySpeed and resource use while executing actions.How many steps are required to complete the task?
Retrieval / Search QualityRelevance and completeness of fetched evidence.Did retrieval surface the right context at decision time?
Transformation QualityAccuracy of transformations, summaries, and structuring.Is transformed output faithful to source inputs?
Selection / RankingQuality of prioritization among available options.Does ranking surface the highest-value output first?
Loop EfficiencyIterations needed to reach a valid stop condition.Are loops converging quickly or cycling?
Reliability & SafetyError, failure, and policy compliance behavior.How often does the system fail or violate constraints?
Cost & LatencyTime and spend required per successful outcome.Is performance acceptable at target load and budget?
Outcome AlignmentMatch between system outputs and business outcomes.Do better outputs produce better real outcomes?

4. Primary System Metric

  • Outcome-aligned task success rate
  • Definition: share of loops that reach a valid, useful end state.
  • Why it matters: ties system behavior to delivered value.
  • Target: improve quality without increasing failure cost.

5. Secondary Metrics

  • Planner first-pass acceptance rate
  • Execution completion rate
  • Evaluation pass rate
  • User acceptance / downstream adoption rate

6. Diagnostic Metrics

  • Average loop count per completed task
  • Retry and rollback rate
  • Tool-call error distribution
  • Stage-level drop-off by Planner, Execution, and Evaluation

7. Guardrail Metrics

  • Policy violation rate
  • Unsafe output rate
  • P95 latency ceiling
  • Cost-per-success ceiling

8. Metric Relationships

Better planner decisions → Better execution → Better outputs → Higher-quality outcomes → Lower cost

9. Measurement Strategy

Data sources

  • State transition logs
  • Planner and evaluation traces
  • Execution events and tool telemetry

Tracking approach

  • Stage-level instrumentation
  • Loop-level attribution
  • Outcome-linked cohort analysis

10. Success Criteria

  • Primary metric improves against baseline.
  • Secondary metrics move in the same direction.
  • Guardrails remain within operating thresholds.

11. Agentic System Performance Metrics

A. Decision Quality

  • Plan correctness
  • Decision reversals
  • Goal alignment score

B. Reliability

  • Task failure rate
  • Execution exception rate
  • Recovery success rate

C. Latency & Cost

  • Time to successful completion
  • Cost per successful loop
  • Token / compute per outcome

D. Guardrails

  • Policy breach rate
  • Unsafe action prevention rate
  • Compliance check pass rate

12. Evaluation Strategy

Offline

  • Scenario suites by system state complexity
  • Regression checks on critical tasks
  • Adversarial and boundary-case evaluation

Online

  • Shadow and staged rollouts
  • Outcome lift by cohort
  • Loop-level alerts on reliability and cost drift

13. System Interfaces (Optional)

1. Planner Service

  • Input: current state, objectives, constraints
  • Output: ranked plan candidates
  • Role: choose a feasible next path

2. Execution Layer

  • Input: selected plan and tool permissions
  • Output: action traces and result artifacts
  • Role: execute reliably with clear observability

3. Evaluation Engine

  • Input: outputs, expected criteria, policy checks
  • Output: pass/fail signals with scores
  • Role: enforce quality and safety gates

4. Decision Controller

  • Input: evaluation results and system budget state
  • Output: continue, revise, retry, or stop decision
  • Role: optimize loop quality, cost, and completion time

5. Analytics Layer

  • Input: events, traces, metrics, and outcome labels
  • Output: dashboards, alerts, and root-cause views
  • Role: drive iteration with measurable evidence

When to use this Framework

Works well for

  • Multi-step agentic products with measurable outcomes
  • Systems where loop behavior drives product value
  • Teams optimizing quality, reliability, and cost together

Not ideal for

  • Single-step deterministic workflows
  • Concept-stage products without stable outcomes
  • Systems lacking basic event instrumentation
