Analytics Framework

A structured approach to measuring product outcomes and AI system performance.

Most effective when linking features → behavior → outcomes.

1. Objective

  • Goal
  • North Star Metric

2. User Funnel

Stage User Action Description
DiscoveryFinds product/optionsUser becomes aware of relevant choices.
ShortlistingFilters and saves optionsUser narrows options based on needs.
EvaluationCompares detailsUser validates fit, quality, and tradeoffs.
DecisionSelects an optionUser commits to a preferred choice.
CommitmentCompletes key actionUser finalizes the intended outcome.

3. Product Metrics

Metric Type Definition Why it matters Direction
Decision conversion ratePrimary% of users reaching commitmentDirect measure of value deliveredIncrease
Evaluation completion rateSecondary% users finishing comparison flowShows healthy mid-funnel behaviorIncrease
Regenerate rateDiagnostic% sessions with repeated retriesSignals output quality frictionDecrease
Error rateGuardrail% failed or invalid responsesProtects reliability and trustDecrease

Metric Types

  • Primary → core success metric
  • Secondary → supporting outcomes
  • Diagnostic → helps debug behavior
  • Guardrail → ensures no negative impact

13. System Interfaces (Optional)

1. Ranking Service

  • Input: User context, item features
  • Output: Ordered candidates
  • Role: Prioritize relevant options

2. Comparison Builder

  • Input: Ranked options, attributes
  • Output: Structured comparison view
  • Role: Support informed evaluation

3. LLM Service

  • Input: Prompt, context, constraints
  • Output: Natural-language guidance
  • Role: Explain and summarize tradeoffs

4. Validation Layer

  • Input: LLM outputs, policy rules
  • Output: Safe, compliant response
  • Role: Enforce reliability and safety

5. Logging / Analytics

  • Input: Events across the full funnel
  • Output: Dashboards and diagnostics
  • Role: Measure outcomes and iterate

When to use Analytics Framework

Works well for

  • AI-enabled products
  • Funnel-driven products
  • Decision-support systems
  • Systems with measurable outcomes

Not ideal for

  • Pure exploratory systems
  • Early ideation phases
  • Highly creative workflows
